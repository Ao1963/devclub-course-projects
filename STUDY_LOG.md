## [2026-04-27] - Project Architecture & Directory Documentation
**Topics:** #Workflow #Documentation #DirectoryTree #BestPractices #KebabCase

### 💡 Key Learnings

- **Directory Tree Representation:** Mastered the professional way to document folder structures using visual symbols (`├──`, `└──`, `│`). This improves communication in `README.md` files and technical documentation.
- **Naming Conventions:** Standardized project naming using **kebab-case** (e.g., `css-intermediate-project/`). This is a critical industry standard for web servers and SEO compatibility.
- **Environment Tools:** 
    - Learned how to use the macOS Terminal integrated with VS Code.
    - Installation and usage of the `tree` command via Homebrew to automate structural visualization.
- **Professional Hierarchy:** Learned how to organize assets (images/styles) separately from the main HTML structure to ensure scalability.

### 🎯 Professional Impact
Implementing these standards transforms a "learner's folder" into a "professional repository." It demonstrates attention to detail, maintainability, and readiness for collaborative environments (Git/GitHub).
